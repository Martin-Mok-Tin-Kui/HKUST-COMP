Questions and Answers
  • How and when to use the keyword this?
    • this is a reference to the current object. Using this keyword, you can refer to any member of the current object from within an instance or a constructor.

  • How and when to use the keyword super?
    •
    • super is used to refer the immediate parent class object. super can be used to refer the instance variables and invoke methods of the immediate parent class. super() can be used to invoke the constructor of the immediate parent class.

  • Why should I override the toString() function?
    •
    • Suppose we want to print our Post object using System.out.println(post). If we don't override toString(), its default behavior is to print "class name@ hash code", so we'll get something like "Post@12345f", which doesn't really make sense!
    • So, we override toString() to return proper and meaningful output. For example, System.out.println(post) can print something like "This post has ID xxx and Content yyy" based on the way we implement toString().

  • Why should I override the equals() and hashCode() functions?
    •
    • Suppose we have two User objects u1 and u2, which have the same user IDs and user names. So we consider u1 and u2 to be the same user.
    • But when we test if (u1.equals(u2)), it returns false. Why? Because the default behavior of equals() compares the objects' locations in memory, which are different for u1 and u2.
    • So, we want to override equals() to say, if both User objects have the same user IDs and same user names, then they are equal  (i.e., return true)
    • Now, if we override equals(), we must also override hashCode(). This ensures **if two objects are equal according to the equals(Object) method, then calling the hashCode() method on each of the two objects must produce the same integer result**, which is the general rule defined in hashCode contract. Otherwise we'll violate this rule because the default hashCode(), if not overridden, returns distinct integers for distinct objects.
